This guide covers how to cleanly remove NextRoute from your system.

---

## Quick Uninstall (v3.6.2+)

NextRoute provides two built-in scripts for clean removal:

### Keep Your Data

```bash
npm run uninstall
```

This removes the NextRoute application but **preserves** your database, configurations, API keys, and provider settings in `~/.nextroute/`. Use this if you plan to reinstall later and want to keep your setup.

### Full Removal

```bash
npm run uninstall:full
```

This removes the application **and permanently erases** all data:

- Database (`storage.sqlite`)
- Provider configurations and API keys
- Backup files
- Log files
- All files in the `~/.nextroute/` directory

> ⚠️ **Warning:** `npm run uninstall:full` is irreversible. All your provider connections, combos, API keys, and usage history will be permanently deleted.

## Manual Uninstall

### NPM Global Install

```bash
# Remove the global package
npm uninstall -g nextroute

# (Optional) Remove data directory
rm -rf ~/.nextroute
```

### pnpm Global Install

```bash
pnpm uninstall -g nextroute
rm -rf ~/.nextroute
```

### Docker

```bash
# Stop and remove the container
docker stop nextroute
docker rm nextroute

# Remove the volume (deletes all data)
docker volume rm nextroute-data

# (Optional) Remove the image
docker rmi diegosouzapw/nextroute:latest
```

### Docker Compose

```bash
# Stop and remove containers
docker compose down

# Also remove volumes (deletes all data)
docker compose down -v
```

### Electron Desktop App

**Windows:**

- Open `Settings → Apps → NextRoute → Uninstall`
- Or run the NSIS uninstaller from the install directory

**macOS:**

- Drag `NextRoute.app` from `/Applications` to Trash
- Remove data: `rm -rf ~/Library/Application Support/nextroute`

**Linux:**

- Remove the AppImage file
- Remove data: `rm -rf ~/.nextroute`

### Source Install (git clone)

```bash
# Remove the cloned directory
rm -rf /path/to/nextroute

# (Optional) Remove data directory
rm -rf ~/.nextroute
```

## Data Directories

NextRoute stores data in the following locations by default:

| Platform      | Default Path                  | Override                  |
| ------------- | ----------------------------- | ------------------------- |
| Linux         | `~/.nextroute/`               | `DATA_DIR` env var        |
| macOS         | `~/.nextroute/`               | `DATA_DIR` env var        |
| Windows       | `%APPDATA%/nextroute/`        | `DATA_DIR` env var        |
| Docker        | `/app/data/` (mounted volume) | `DATA_DIR` env var        |
| XDG-compliant | `$XDG_CONFIG_HOME/nextroute/` | `XDG_CONFIG_HOME` env var |

### Files in the data directory

| File/Directory       | Description                                       |
| -------------------- | ------------------------------------------------- |
| `storage.sqlite`     | Main database (providers, combos, settings, keys) |
| `storage.sqlite-wal` | SQLite write-ahead log (temporary)                |
| `storage.sqlite-shm` | SQLite shared memory (temporary)                  |
| `call_logs/`         | Request payload archives                          |
| `backups/`           | Automatic database backups                        |
| `log.txt`            | Legacy request log (optional)                     |

## Verify Complete Removal

After uninstalling, verify there are no remaining files:

```bash
# Check for global npm package
npm list -g nextroute 2>/dev/null

# Check for data directory
ls -la ~/.nextroute/ 2>/dev/null

# Check for running processes
pgrep -f nextroute
```

If any process is still running, stop it:

```bash
pkill -f nextroute
```
